# CBQR - 차량관련 방문 업체 현황 파악 시스템

## 개요
이 시스템은 각 경찰서별로 차량 관련 업체 정보를 체계적으로 관리하고, QR 코드를 통해 빠르게 접근할 수 있도록 구성된 웹 애플리케이션입니다.

## 파일 구성 방법

### 1. 메인 페이지 (index.html)
각 경찰서별 메인 페이지는 다음 메뉴로 구성:
- 사고 났어요 → accident.html
- 차량이 고장났어요 → breakdown.html
- 주유소 어디로 가죠!? → gas-main.html
- 세차장은 어디죠? → carwash.html
- 담당업체는 어디죠? → companies.html

**⚠️ 중요: 버튼 순서 규칙**
- **반드시 "사고 났어요"가 "차량이 고장났어요"보다 먼저 나와야 함**
- 기존 폰에서 틀어버리지 않도록 순서 준수 필수
- 표준 순서: 사고 → 고장 → 주유소 → 세차장 → 담당업체

**⚠️ 중요: 메뉴 구성 필수 요소**
- **5개 메뉴 모두 필수**: 사고, 고장, 주유소, 세차장, 담당업체
- **세차장 메뉴 필수**: 모든 경찰서에 세차장 메뉴가 있어야 함
- **메뉴 제목 통일**: 정확한 제목으로 표기 (예: "세차장은 어디죠?")

**메뉴별 상세 구성**:
1. **사고 났어요** (accident.html)
   - 사고 처리 기본 절차
   - 견인, 타이어, 이동 등 세부 절차

2. **차량이 고장났어요** (breakdown.html)
   - 기능 미작동, 외형 변형, 배터리 등
   - 관서별 정비소 정보

3. **주유소 어디로 가죠!?** (gas-main.html)
   - 관서별 주유소 정보
   - S-OIL 우선, 타 정유회사 가능

4. **세차장은 어디죠?** (carwash.html)
   - 관서별 세차장 정보
   - 손세차, 자동세차 구분

5. **담당업체는 어디죠?** (companies.html)
   - 정기점검, 월간점검, 타이어, 썬팅 등
   - 각 업체별 상세 정보

### 2. 차량 기능 미작동 페이지 구성

#### 2-1. 구성 방식 구분

**A. 충북청 방식 (한 페이지에 모든 관서)**
- `breakdown-function.html`에 모든 관서의 처리 절차를 한 페이지에 나열
- 관서가 적거나 단순한 구조의 경우 사용

**B. 흥덕서 방식 (관서별 버튼 → 상세 페이지)**
- `breakdown-function.html`: 관서별 버튼 메뉴
- 각 버튼 클릭 시 해당 관서들의 상세 페이지로 이동
- 관서가 많거나 복잡한 구조의 경우 사용

#### 2-2. 관서별 타이틀 표기 규칙
- **본서**: "경찰서 본서"로 표기
- **지구대**: "[지구대명]지구대" (예: 강저지구대, 중앙지구대)
- **파출소**: "[파출소명]파출소" (예: 용두파출소, 봉양파출소)

## 각 경찰서별 수정 가이드

### 새로운 경찰서 추가 시
1. **폴더 생성**: 경찰서명으로 폴더 생성 (예: `newpolice/`)
2. **CSV 파일 확인**: 해당 경찰서의 CSV 파일에서 섹션별 정보 추출
3. **차량담당자 정보 확인**: 섹션 1에서 담당자 정보 확인
4. **업체별 정보 확인**: 각 섹션에서 해당 경찰서의 업체 정보 확인
5. **구성 방식 결정**: 관서 수와 복잡도에 따라 충북청 방식 또는 흥덕서 방식 선택
6. **파일 생성**: 기존 파일들을 복사하여 수정

### 기존 경찰서 수정 시
1. **CSV 파일 업데이트**: 최신 정보로 CSV 파일 업데이트
2. **차량담당자 정보 수정**: 섹션 1의 정보로 모든 연락처 수정
3. **업체 정보 수정**: 각 섹션의 정보로 해당 업체 정보 수정
4. **연락처 통일**: 모든 파일에서 차량담당자 연락처 통일
5. **관서별 타이틀 표기 확인**: 본서는 "경찰서 본서", 지구대/파출소는 풀네임으로 표기

## 주의사항
- 각 경찰서별로 차량담당자 정보가 다르므로 반드시 해당 CSV 파일 확인
- 업체 정보는 CSV 파일의 최신 정보로 업데이트
- 연락처는 클릭 가능한 링크로 구성 (`tel:` 프로토콜 사용)
- 주소는 네이버 지도 링크로 연결
- 모든 파일에서 일관된 형식과 스타일 유지
- 관서별 타이틀 표기 규칙 준수 (본서, 지구대, 파출소 구분)
